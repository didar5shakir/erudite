# План разработки Erudite

## О проекте

Erudite — веб-приложение (PWA) для проверки эрудиции через узнавание известных личностей.

**Позиционирование:** "Тест эрудиции — how many historical figures do you know?"

**Аудитория:** мировая.

**Цель:** серьёзный масштабируемый продукт, сотни тысяч пользователей. Не быстрый прототип.

## Ключевые решения

- **Формат:** веб-сайт (PWA) + Android-приложение через Capacitor.
- **Две механики:** свайпы "знаю / не знаю" + 5 проверочных вопросов в конце для честности результата.
- **Стек:** Next.js + TypeScript + Tailwind CSS + Supabase + Vercel.
- **Без фото на старте** — текстовые карточки. Пересматривается после тестов.
- **База:** 10–15 тысяч фигур из Pantheon + 300–500 локальных.

## Языки

**На старте:**
- **English** — основной, мировой охват
- **Русский** — быстрая обратная связь, 250 млн носителей
- **Қазақ** — дифференциатор, PR-крючок, пустая ниша

**Архитектура:** каждая фигура имеет канонический ID + локализованные поля. Добавление языка = добавление полей, не переписывание кода.

**Позже:** испанский, португальский, арабский, индонезийский — по метрикам, по одному в месяц после запуска.

## Этап 4.5 — AI-инъекции локальных фигур

### Цель

Добавить в базу 500–700 фигур, которых нет в Wikipedia с достаточным охватом.

### Состав

- 200–300 казахстанских (Imanbek, Скриптонит, Tole Bi, Қазыбек би, Айтеке би и др.)
- 100–150 узбекских
- Аналогично для Турции, Украины

### Процесс

- Верификация через Wikidata API
- Ручная проверка списков
- Добавление в figures_top_final.csv с пометкой `inclusion_source='ai_injection'`

## Бюджет и маркетинг

**Стартовый бюджет:** до $1000 на маркетинг и инструменты. Может расти после первой прибыли.

**Стратегия:** органический рост через виральность и SEO. Платный таргет — точечно, для усиления уже работающих органических моментов.

**Ключевые точки:**
- Шеринг и дерево эрудиции на экране результата — **основа маркетинга**, не приятный бонус
- Слабый результат должен быть так же шерабельным, как сильный
- SEO-страницы для каждой из 15 000 фигур (Этап 6-7) — главный бесплатный канал
- Платформы запуска: Reddit (r/quiz, r/history), Hacker News, Product Hunt, Threads

**Английский язык:** перед публичным запуском (Этап 8) — обязательная вычитка нативным редактором (бюджет $50-100 на Upwork).

|---|---|---|
| global | 25 000 | top по global_score |
| ru_quota | 2 000 | top по ru_score, не в global |
| kz_quota | 2 000 | top по kz_score, не в global ∪ ru |
| hpi_quota | 1 000 | top по hpi, не в остальных |

### Формула скоринга V5

**PV_raw** = 0.45 × log1p(pv_total) + 0.45 × log1p(pv_stable) + 0.10 × log1p(pv_peak)

**CAP на пики:** 5 × median по месяцам перед расчётом метрик

**HPI-бустер** (вес 0.30) для всех региональных рейтингов

**Региональные веса языков:**
- KZ: kk×0.50 + ru×0.30 + en×0.15 + other×0.05/8
- RU: ru×0.55 + en×0.25 + kk×0.05 + other×0.15/8
- CIS: ru×0.55 + kk×0.15 + en×0.10 + tr×0.07 + other×0.13/8 (рассчитан, в продукте не используется)

**final_priority_score** = max(global_score, 0.92×ru_score, 0.92×kz_score, 0.88×hpi_percentile)

### Рабочие колоды

- **Global колода** — все 30 000, отсортированы по global_score
- **RU колода** — все 30 000, отсортированы по ru_score
- **KZ колода** — все 30 000, отсортированы по kz_score
- ru_quota / kz_quota / hpi_quota — это гарантия попадания в базу, не отдельная колода

### Известные ограничения

- **Jesus / Muhammad** — попали в базу, но позиции в global_rank ниже ожидаемых (33k/32k по global до финального отбора); Wikipedia разбивает трафик по разным страницам для этих имён
- **Очень мало людей после 2000 года** — 256 из 30 000 (< 1%)
- **Гендерный перекос** — 79% M / 21% F среди записей с известным полом; 22% без данных
- **Локальные KZ-фигуры** — Imanbek, Tole Bi, Qazybek Bi, Aiteke Bi отсутствуют в Wikipedia достаточно; будут добавляться через AI-инъекцию позже

## Самообучение алгоритма

Erudite изначально стартует со статической формулы узнаваемости:
Wikipedia pageviews + Pantheon HPI + региональные веса + квоты.

Но долгосрочная цель — перейти к crowd-calibrated модели, где реальные ответы пользователей постепенно уточняют сложность фигур.

### Этап 5 — адаптация внутри сессии пользователя

- IRT-подобная модель оценивает уровень пользователя по ответам.
- Алгоритм подбирает следующие карточки ближе к уровню пользователя.
- Отдельно учитываются измерения:
  - общий уровень
  - регион
  - категория
  - эпоха
- В MVP можно начать с простой эвристики, без полноценной IRT.

### Этап 8+ — калибровка по aggregate-ответам

После публичного запуска и накопления данных:

- сохранять ответы пользователей:
  - person_id
  - answer: know / heard / dont_know
  - locale
  - selected_region
  - deck
  - category
  - epoch
  - response_time
- считать по каждой фигуре:
  - know_rate
  - heard_rate
  - unknown_rate
  - recognition_score_real
- пересчитывать реальные сложности каждые 1–2 недели.

В начале использовать осторожную смесь:

real_difficulty = 0.80 × initial_model_difficulty + 0.20 × crowd_difficulty

По мере накопления данных вес crowd_difficulty можно повышать.

### Этап 10+ — региональная и crowd-калибровка

- разные difficulty для разных регионов и языков
- кросс-региональная калибровка
- добавление новых фигур через user suggestions

Важно:
AI-инъекция и crowd-learning решают разные задачи:
- AI-инъекция добавляет локальных людей, которых нет в базе.
- Crowd-learning калибрует их реальную узнаваемость после попадания в базу.
